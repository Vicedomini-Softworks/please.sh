please.sh

A polite sudo wrapper with built-in security — command validation, denylist, dry-run, confirmation, and timeout protection.

🙏 What Is please.sh?

please.sh is an open-source shell function that replaces sudo with a safer, friendlier alternative. Every command is validated against a configurable denylist before execution, and optional dry-run and confirmation modes help prevent costly mistakes.

Instead of typing sudo command, you type please command — and please.sh handles the rest:

  1. 🔎 Command Validation — Checks the command exists before calling sudo
  2. 🛑 Denylist Protection — Dangerous commands like rm are blocked by default
  3. 👁️ Command Preview — Shows ⚠️ About to run: sudo ... before executing
  4. 🧪 Dry-Run Mode — Preview without executing: please --dryrun apt update
  5. ✅ Confirmation Prompts — Require typing YES before running risky commands
  6. ⏰ Timeout Protection — Kill hanging commands automatically with PLEASE_TIMEOUT
  7. ✨ Clear Feedback — Know instantly if a command succeeded or failed

It's sudo with manners — and a safety net.

🚀 Usage

Basic Usage

Replace sudo with please:

please apt update
$ please apt update
⚠️ About to run: sudo apt update
[sudo] password for you:
...
✨ Success!

Dry-Run Mode (Preview Without Executing)

Use the --dryrun flag (or -d) to see what would happen:

please --dryrun apt update
$ please --dryrun apt update
⚠️ About to run: sudo apt update
➡️ Dry-run mode enabled. Not executing.

Denylist in Action: Blocking Dangerous Commands

By default, rm is blocked. This prevents accidental rm -rf / disasters:

$ please rm -rf /
❌ Command blocked by safety policy: rm
This command is in the deny list.

🔒 Security Features

🛑 Denylist

Block dangerous commands like rm, dd, and shred. Configurable via the PLEASE_DENYLIST environment variable or ~/.please.config.

✅ Allowlist

Restrict please to only approved commands. Set PLEASE_ALLOWLIST="apt,systemctl" for locked-down environments or CI pipelines.

🔍 Dry-Run Mode

Preview the full sudo command without executing. Use the --dryrun flag or set PLEASE_DRYRUN=1 to enable for all commands.

👆 Confirmation Prompt

Require typing YES before any command runs. Use --confirm per-command or set PLEASE_CONFIRM=1. Skips automatically in non-interactive mode.

⏰ Timeout Protection

Prevent hanging commands with PLEASE_TIMEOUT=30. Automatically kills commands that exceed the limit. Falls back gracefully if the timeout utility isn't available.

🔎 Command Validation

Checks if the command exists on your system before running sudo, saving you from cryptic errors and providing clear feedback.

⚙️ Configuration

Environment Variables

Set these in your ~/.zshrc or ~/.bashrc for persistent configuration:

export PLEASE_CONFIRM=1       # Require confirmation for every command
export PLEASE_DRYRUN=0        # Disable dry-run by default
export PLEASE_TIMEOUT=30      # Kill commands after 30 seconds
export PLEASE_DENYLIST="rm,dd,shred"  # Block these commands
export PLEASE_ALLOWLIST=""    # Empty = allow any (except denylist)

Config File (~/.please.config)

Create a ~/.please.config file for persistent settings:

# ~/.please.config
PLEASE_DENYLIST="rm,dd,shred,chmod"
PLEASE_ALLOWLIST="apt,systemctl,service"
PLEASE_CONFIRM=1
PLEASE_DRYRUN=0
PLEASE_TIMEOUT=30

Command-Line Flags

please --help              # Show full help with all options
please --dryrun apt update # Preview without executing
please -d apt update       # Same as --dryrun (short form)
please --confirm rm file   # Require YES confirmation
please -c rm file          # Same as --confirm (short form)

🤔 How It Works

please.sh is a simple shell function that wraps sudo with multiple safety layers:

  1. Validates the command exists on your system
  2. Checks the command against your denylist and allowlist
  3. Previews the full sudo command: ⚠️ About to run: sudo apt update
  4. Optionally pauses for dry-run or confirmation
  5. Executes with timeout protection (if configured)
  6. Reports success or failure with the exit code
please() {
  command -v "$1"        # 1. Does the command exist?
  check denylist         # 2. Is it allowed?
  echo "About to run"    # 3. Show what's happening
  read confirmation      # 4. Ask for approval (optional)
  timeout ... sudo "$@"  # 5. Execute with guardrails
  report status          # 6. Tell you the result
}

The "$@" syntax preserves all your arguments exactly, so please apt update runs sudo apt update with perfect fidelity.

🐛 Troubleshooting

Q: please: command not found

A: Make sure you sourced the script or restarted your shell. Run source ~/.zshrc or open a new terminal.

Q: Still asking for the sudo password every time

A: That's sudo's default behavior, not please.sh. Configure /etc/sudoers if you want passwordless sudo for specific commands.

Q: Can I use this with fish, zsh, or bash?

A: Yes! please.sh works in any POSIX-compatible shell that supports functions — bash, zsh, dash, and more.

Q: rm is blocked — how do I use it?

A: Use sudo rm directly, or remove rm from PLEASE_DENYLIST in ~/.please.config.

Q: The confirmation prompt hangs in scripts

A: please.sh automatically detects non-interactive mode (piped input) and skips the confirmation prompt. Set PLEASE_CONFIRM=0 to disable permanently.

Q: How do I contribute or report a bug?

A: Open an issue or pull request on GitHub. The project is MIT licensed and welcomes contributions.
