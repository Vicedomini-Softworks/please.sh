please.sh

A sudo wrapper that asks permission first (from you, to your terminal)

🙏 What is this?

please is a shell function that makes your terminal more polite and safer. Instead of just running sudo command, you now run please command and your terminal will:

  1. ⚠️ Show the exact command: "About to run: sudo ..." before executing
  2. ✅ Validate the command exists and block dangerous commands like rm
  3. ✨ Say "Success!" if it works
  4. 🛡️ Protect with dry-run, confirmation prompts (--confirm), and timeouts
  5. ❌ Say "that didn't work" if it fails (with helpful tips)

It's sudo, but with manners and a safety net.

🚀 Usage

Instead of this:

sudo apt update

Do this:

please apt update
$ please apt update
⚠️ About to run: sudo apt update
[sudo] password for you:
...
✨ Success!

With safety features:

$ please --dryrun apt update
⚠️ About to run: sudo apt update
➡️ Dry-run mode enabled. Not executing.
$ please rm -rf /
❌ Command blocked by safety policy: rm
This command is in the deny list.

🔒 Security Features

🛑 Deny List

Block dangerous commands like rm, dd, shred by default. Configure via PLEASE_DENYLIST.

✅ Allow List

Restrict to only approved commands. Set PLEASE_ALLOWLIST="apt,systemctl" for locked-down environments.

🔍 Dry-Run

Preview commands without executing: please --dryrun apt update or set PLEASE_DRYRUN=1.

👆 Confirmation

Require typing YES before running: please --confirm rm file or set PLEASE_CONFIRM=1.

⏰ Timeout

Prevent hanging commands with PLEASE_TIMEOUT=30. Falls back gracefully if timeout isn't available.

🔎 Command Validation

Checks if the command exists before running sudo, with clear error messages.

🤔 How does it work?

It's a shell function that wraps sudo with validation and safety layers:

  1. Validates the command exists on your system
  2. Checks against configured allowlist/denylist
  3. Previews the command: ⚠️ About to run: sudo apt update
  4. Protects with optional dry-run, confirmation (Type 'YES' to execute), and timeout
  5. Executes safely with "$@" preserving all arguments
please() {
  command -v "$1"       # 1. exists?
  check denylist        # 2. safe?
  echo "About to run"   # 3. preview
  read confirmation     # 4. confirm (optional)
  sudo "$@"             # 5. execute
}

The "$@" preserves all your arguments, so please apt update becomes sudo apt update with perfect fidelity.

A ~/.please.config file is sourced automatically if present to set your safety preferences.

🛠️ Customization

🎨 Message Customization

Edit ~/.please.sh to change the output messages:

# Make it sassier
please() {
  [ $# -eq 0 ] && echo "🙄 I need a command, genius" && return 1
  echo "🥺 *puppy eyes* Running with sudo..."
  sudo "$@" && echo "🎉 you're welcome" || echo "💀 oopsie"
}

⚙️ Config File (~/.please.config)

# Block additional dangerous commands
PLEASE_DENYLIST="rm,dd,shred,chmod"

# Only allow safe commands (lockdown mode)
PLEASE_ALLOWLIST="apt,systemctl,service"

# Always confirm before executing
PLEASE_CONFIRM=1

# Preview everything without running
PLEASE_DRYRUN=1

# Kill commands that hang for 30+ seconds
PLEASE_TIMEOUT=30

🌍 Environment Variables

Set these in your ~/.zshrc or ~/.bashrc for persistent config:

export PLEASE_CONFIRM=1
export PLEASE_DRYRUN=0
export PLEASE_TIMEOUT=30
export PLEASE_DENYLIST="rm,dd,shred"
export PLEASE_ALLOWLIST=""

🐛 Troubleshooting

Q: please: command not found

A: Make sure you sourced the script or restarted your shell

Q: Still asking for password every time

A: That's sudo's behavior, not please. Configure /etc/sudoers if you want passwordless sudo

Q: Can I use this with fish/zsh/bash?

A: Yes! Works in any POSIX shell that supports functions

Q: rm is blocked, how do I use it?

A: Use sudo rm directly, or remove rm from PLEASE_DENYLIST in your config

Q: Confirmation prompt hangs in scripts

A: please detects non-interactive mode and skips confirmation automatically
